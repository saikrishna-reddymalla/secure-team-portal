# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.x     | Yes       |

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Use GitHub's private vulnerability reporting in the **Security** tab.

Please include:
- summary of the issue
- affected endpoint or file
- reproduction steps
- proof of concept
- impact

## Response Process

We aim to:
- acknowledge reports within 7 days
- validate the issue
- prepare a fix
- publish an advisory after remediation

## Scope

Examples of valid reports:
- authentication bypass
- broken access control
- IDOR
- stored XSS
- CSRF
- SQL injection
- file upload bypass
- privilege escalation

## Recognition

Researchers may be credited in:
- SECURITY-HALL-OF-FAME.md
- release notes
