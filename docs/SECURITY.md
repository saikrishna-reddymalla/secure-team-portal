# Security Policy

## Supported Versions

| Version | Supported |
|-------|-----------|
| 1.x   | Yes |

## Reporting a Vulnerability

Please **do not open public issues for security vulnerabilities**.

Instead use GitHub's **Report a Vulnerability** feature in the Security tab.

Include:

- Summary of the issue
- Steps to reproduce
- Affected endpoints/files
- Proof of concept
- Possible impact

## Response Process

We aim to:

- acknowledge reports within 7 days
- confirm the vulnerability
- create a patch
- publish a security advisory

## Scope

Examples of valid reports:

- Authentication bypass
- Broken access control
- IDOR
- Stored XSS
- CSRF
- SQL injection
- File upload bypass
- Privilege escalation

## Recognition

Researchers may be credited in:

- SECURITY-HALL-OF-FAME.md
- release notes
